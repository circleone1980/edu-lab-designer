## 2. AI 能力组件

### 2.1 大模型（LLM）

| 用途 | LARP 课 | 推演课 |
|------|--------|-------|
| 文本生成 | 角色卡、线索、AI 对话、内心独白 | 旁白、追问、推演、反思提示 |
| 角色扮演 | AI NPC 多角色对话 | AI 旁白/追问官/推演引擎 |
| 个性化 | 基于行为生成个性化回应 | 基于投票生成个性化提示 |

### 2.2 RAG 检索增强

| 用途 | 说明 |
|------|------|
| 历史准确性 | 确保 AI 输出的历史事实有出处 |
| 资料包生成 | 从学科文献中提取三视角材料 |
| 知识关联 | 自动关联相关理论和事件 |

### 2.3 知识图谱

| 用途 | 说明 |
|------|------|
| 角色关系 | 角色之间的社会关系网络 |
| 事件因果 | 历史事件的前因后果链 |
| 理论关联 | 情节到理论的映射关系 |

### 2.4 多 Agent 对话

| 用途 | 说明 |
|------|------|
| AI 角色协同 | 多个 AI NPC 同时在场时的对话协调 |
| 情节分支管理 | 根据学生选择动态调整剧情 |
| 实时监控 | 监控讨论进程，适时引导 |

### 2.5 情感分析

| 用途 | 说明 |
|------|------|
| 学生情绪监控 | 分析学生输入文本的情感倾向 |
| 反思日记分析 | 分析反思的深度和主题 |
| 教学效果评估 | 量化学生的情感触点 |

## 3. 硬件配置映射

### 3.1 标准实验室配置

| 设备 | 数量 | LARP 用途 | 推演用途 |
|------|------|----------|---------|
| 大屏/投影 | 1 | 场景画面、线索展示 | 投票结果、推演可视化 |
| 六边形桌 | 1 | 5-6人主角围坐 | 全班分组或独立操作 |
| 电脑终端 | 6 | 角色卡、线索、投票 | 资料包、投票、日记 |
| VR 头显 | 6（可选） | 沉浸场景体验 | 场景回顾 |
| 教师控制台 | 1 | 进度控制、数据监控 | 投票控制、数据监控 |
| 音响系统 | 1套 | 环境音、AI 语音 | 旁白、音效 |
| 数据服务器 | 1 | 本地模型+数据 | 同左 |
| 学生手机/平板 | 自带 | — | 扩展终端（推演全班参与） |

### 3.2 空间布局原则

- 六边形桌居中，大屏在正前方
- 终端围绕六边形桌摆放
- 教师控制台在大屏旁侧
- VR 区域与桌面区域有足够间距
- 灯光可分区控制（沉浸模式/正常模式）

## 4. 软件功能映射

### 4.1 LARP 引擎功能

| 功能 | 说明 | 技术实现 |
|------|------|---------|
| 角色卡分发 | 按角色分配到对应终端 | 用户管理+权限控制 |
| 线索推送 | 定时或触发式推送个人线索 | WebSocket 实时推送 |
| AI NPC 对话 | AI 角色与学生实时互动 | LLM + TTS/ASR |
| 选择收集 | 各幕选择实时收集 | 表单+数据库 |
| 剧情分支 | 根据多数选择调整后续剧情 | 状态机+分支管理 |
| 进度控制 | 教师控制流程推进 | 教师控制台 |

### 4.2 推演引擎功能

| 功能 | 说明 | 技术实现 |
|------|------|---------|
| 资料包分发 | 三视角资料同步推送到所有终端 | 批量推送+阅读追踪 |
| 投票系统 | 匿名投票、同步揭晓、分类统计 | WebSocket+实时图表 |
| AI 追问 | 根据陈述内容生成追问 | LLM + 追问策略模板 |
| 修正追踪 | 记录立场变化轨迹 | 决策引擎+数据库 |
| 后果推演 | AI 推演每种选择的后果 | LLM + RAG（历史约束） |
| 反思日记 | 学生在终端输入反思文本 | 表单+文本分析 |

## 5. 数据流设计

### 5.1 LARP → 推演数据传递

```
LARP 课数据                     推演课使用
┌──────────────┐               ┌──────────────┐
│ 学生角色      │──── 引用 ───→│ 个性化导入语  │
│ 各幕选择      │──── 对比 ───→│ 对比讨论问题  │
│ 线索查看记录  │──── 参考 ───→│ AI 追问依据   │
│ 最终抉择      │──── 对比 ───→│ 反思提示      │
│ 信件内容      │──── 引用 ───→│ AI 回应       │
└──────────────┘               └──────────────┘
```

### 5.2 教学分析数据流

```
课堂行为数据 → 情感分析 → 参与度评分
                           ↓
选择数据 ──→ 立场分析 ──→ 价值倾向画像
                           ↓
反思文本 ──→ 主题分析 ──→ 认知深度评估
                           ↓
                    综合教学分析报告
```

## 6. 技术选型建议

| 层级 | 推荐方案 | 备选方案 |
|------|---------|---------|
| 大模型 | 本地部署（隐私保护） | API 调用（快速启动） |
| RAG | 向量数据库 + 文档切片 | 关键词检索（低成本） |
| 知识图谱 | Neo4j / 本地图谱 | 关系型数据库（简化） |
| 前端 | Web 应用（浏览器访问） | 桌面应用（离线支持） |
| 通信 | WebSocket（实时） | HTTP 轮询（简化） |
| 数据库 | PostgreSQL + pgvector | MySQL + ElasticSearch |
| 部署 | 本地服务器（教育场景） | 云服务（弹性扩展） |
