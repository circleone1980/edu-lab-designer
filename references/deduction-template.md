# 推演脚本完整模板

> 基于 7 步流程的关键抉择推演脚本模板，适用于所有学科。
> 每个推演总时长约 55 分钟。

---

## 推演标题格式

直接描述核心抉择情境，如：《夜袭前夜》《血路抉择》《联大十字路口》

---

## 一、推演设定

### 必填要素

| 要素 | 说明 | 示例 |
|------|------|------|
| 历史原型 | 与配对 LARP 共享的历史事件 | 1936年4月18日傍晚 |
| 时间 | 决策发生的时间点 | 夜袭前数小时 |
| 地点 | 决策发生的空间 | 云南省祥云县衙 |
| 参与人数 | 全班学生 + 1名教师 + AI | — |
| 核心抉择 | 一句话表述 | 开城迎接红军，还是按省府命令死守城池？ |
| 教育核心 | 一句话概括 | 理解"民心向背"不是抽象概念 |

### 理论锚点

列出 2-3 个本学科核心理论：
- [理论1]：[简述]
- [理论2]：[简述]
- [理论3]：[简述]

### 与配对 LARP 的衔接

> 学生已在《[LARP名称]》中扮演[角色类型]，亲历了[核心事件]。
> 本推演课切换到[决策者]视角——从"经历者"升到"决策者"。
> AI 将在情境导入时引用上堂课经历。

## 三、七步详细交互

### 第0步：情境导入（5分钟）

**大屏开场动画**

```
[画面描述]：从[起点]推近至[核心场景]。
[文字]："[时代背景+核心问题，1-2句]"
[动态元素]：[地图/关键视觉]
```

**AI 旁白**：

> **AI·旁白**（[语气描述]）：
>
> [情境描述，2-3段]
> - 第1段：时间、地点、大背景
> - 第2段：决策者面临的具体困境
> - 第3段：两份对立的情报/观点

**配对衔接**：
> AI 引用上堂 LARP 经历："上一堂课，你们扮演了[角色]。今天，你们是[决策者]——[核心挑战]。"

### 第2步：抉择投票（5分钟）

**终端弹出核心抉择**：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
你的抉择

[核心问题，1-2句]

你的选择是——

A. [选项A名称]
——[简要说明，含收益和风险]

B. [选项B名称]
——[简要说明，含收益和风险]

C. 自拟方案
——如果你认为A和B都不够好，请提出你自己的方案。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**同步揭晓**：

```
┌─────────────────────────────────┐
│      投票结果 · 同步揭晓          │
│                                 │
│  A. [选项A]    ████████  XX%    │
│  B. [选项B]    ████      XX%    │
│  C. 自拟方案   ██████    XX%    │
│                                 │
│  六边形桌 vs 全班（分列显示）     │
└─────────────────────────────────┘
```

---

### 第3步：论证（10分钟）

**AI 追问官出场**：

> **AI·追问官**：
>
> 投票结果已揭晓。不同选择背后是不同的价值判断。请各派代表陈述理由，之后我会追问。

**学生陈述**（3-4名，每人1-2分钟）

**AI 追问策略**（四种轮换）：

| 策略 | 针对选A | 针对选B | 针对选C |
|------|--------|--------|--------|
| 事实追问 | "你依据什么判断[前提]？" | "你凭什么认为[前提]成立？" | "你的方案在当时的条件下可行吗？" |
| 价值追问 | "你把[X]放在首位，但[Y]呢？" | "你选择了'职责'优先，但[X]呢？" | "你的方案体现了什么价值排序？" |
| 反事实追问 | "如果判断错了，你能承担后果吗？" | "如果[反事实情况]呢？" | "有没有先例可以参考？" |
| 逻辑追问 | [检查论证链条的隐含假设] | [检查因果推理是否成立] | [检查方案的完整性和自洽性] |

**旁观者参与**：全班其他学生在终端投票"哪位同学的陈述最有说服力？"

---

### 第4步：修正（5分钟）

> **AI·追问官**：
>
> 听了其他同学的理由和追问后，你是否需要修正？

终端弹出修正选项：
- "我坚持原选择"
- "我修正为A"
- "我修正为B"
- "我修正为C"

**修正前后对比**：

```
┌──────────────────────────────────┐
│  修正前          →    修正后       │
│  A: XX%               A: XX%     │
│  B: XX%               B: XX%     │
│  C: XX%               C: XX%     │
│                                  │
│  共XX人修正了选择                  │
│  主要变化：[变化趋势]              │
└──────────────────────────────────┘
```

AI 请修正了选择的学生用一句话说明原因。

---

### 第5步：AI 后果推演（10分钟）

> **AI·推演引擎**：
>
> 现在推演每种选择的可能后果。以下推演基于当时的历史条件——你没有上帝视角。

**选择A的推演**：

> **短期后果（1-3天）**：[推演内容]
> **中期后果（1-3个月）**：[推演内容]
> **长期影响**：[推演内容]

**选择B的推演**：

> **短期后果（1-3天）**：[推演内容]
> **中期后果（1-3个月）**：[推演内容]
> **长期影响**：[推演内容]

**选择C的推演**：

> AI 根据学生提出的具体方案逐一分析可行性，标注"有历史依据"和"合理推测"。

**理论对照**：

> **AI·推演引擎**：
>
> 回顾两种选择的推演——
> 选择A的背后是：[理论解析]
> 选择B的背后是：[理论解析]

---

### 第6步：真相揭示（5分钟）

> **AI·旁白**（语气庄重）：
>
> 现在告诉你们，历史真实发生了什么。

大屏播放历史真相（3-5条核心事实）：

1. [真实事实1]
2. [真实事实2]
3. [真实事实3]
4. [真实事实4]

> **AI·旁白**：
>
> [总结性的一句话，点明核心价值]

---

### 第7步：复盘反思（10分钟）

**理论对照表**（大屏自动显示）：

| 推演中的抉择 | 对应理论 | 理论解析 |
|-------------|---------|---------|
| [抉择1] | [理论1] | [解析] |
| [抉择2] | [理论2] | [解析] |
| [抉择3] | [理论3] | [解析] |
| [抉择4] | [理论4] | [解析] |

**讨论问题**（AI 提出 3-4 个）：

> **AI·教师**：
>
> 1. [关于因果的问题]
> 2. [关于价值冲突的问题]
> 3. [反事实假设问题]
> 4. [与 LARP 课的对比问题]："上一堂课你们扮演[角色]时，希望[决策者]怎么做？今天你成了[决策者]，选择变了吗？"

**反思日记**（终端输入，200-300字）：

> **AI 提示**（个性化）：
>
> [根据学生投票选择生成]：你在推演中选择了[XX]，主要理由是[从陈述中提取]。在反思中，你可以思考：
> - 你的选择和真实历史一致吗？
> - 上一堂 LARP 课你的角色经历，如何影响了今天的判断？

## 五、技术实现对应表

| 环节 | 用到的技术 | AI 做什么 |
|------|-----------|---------|
| 情境导入 | 大模型 + RAG + 大屏渲染 | 旁白生成，RAG 确保历史准确 |
| 资料包推送 | RAG + 知识图谱 | 提取三视角资料，图谱关联 |
| 抉择投票 | 决策引擎 + WebSocket | 收集投票，同步揭晓 |
| 理由陈述 | 大模型 + 追问策略 | AI 追问官生成追问 |
| 方案修正 | 决策引擎 | 记录修正轨迹 |
| 后果推演 | 大模型 + RAG + 知识图谱 | 推演后果，标注依据 |
| 真相揭示 | RAG + 大屏 | 展示真实数据 |
| 复盘反思 | 大模型 + 情感分析 | 个性化反思提示 |
| 配对衔接 | 数据层关联 | 读取 LARP 数据，生成对比引导 |

## 六、质量检查清单

- [ ] 包含完整 7 步流程
- [ ] 核心抉择有明确的"A vs B"结构
- [ ] 三视角资料包（A方/B方/第三方）各有 4-5 条材料
- [ ] AI 追问策略完整（四种策略轮换）
- [ ] 后果推演含短/中/长期三个时间尺度
- [ ] 理论映射表完整（每个关键抉择都有对应理论）
- [ ] 技术实现表完整
- [ ] 与配对 LARP 的衔接逻辑清晰
- [ ] 本地/区域元素融入（至少 3 处）
- [ ] 反思日记提示是个性化的（基于学生投票数据）
