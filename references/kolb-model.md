# Kolb 体验式学习模型 · 学科适配指南

---

## 1. 模型概述

David Kolb 的体验式学习循环由四个阶段构成，每个阶段对应不同的教学形式：

```
        ┌──────────────┐
        │  具体体验      │ ← LARP 沉浸课
        │ （Concrete    │    学生扮演角色，情感代入
        │  Experience） │
        └──────┬───────┘
               ↓
        ┌──────────────┐
        │  反思观察      │ ← 间隔消化期（1周）
        │ （Reflective  │    AI 推送个性化反思提示
        │ Observation） │
        └──────┬───────┘
               ↓
        ┌──────────────┐
        │ 抽象概念化     │ ← 推演课
        │ （Abstract    │    学生以自身身份理性判断
        │ Conceptuali-  │
        │ zation）      │
        └──────┬───────┘
               ↓
        ┌──────────────┐
        │  主动实践      │ ← 反思日记 + 课程应用
        │ （Active      │    将判断转化为认知框架
        │ Experimenta-  │
        │ tion）        │
        └──────────────┘
```

---

## 2. 为什么"先 LARP 后推演"

| 顺序 | 逻辑 | 效果 |
|------|------|------|
| **先 LARP** | 情感体验 → 建立情感记忆 | 学生对事件有"亲身经历"的感受 |
| 间隔消化 | 反思沉淀 → 感性过渡到理性 | 不急于判断，让体验沉淀 |
| **后推演** | 理性判断 → 用理论分析体验 | 带着情感记忆做理性分析，判断更深刻 |

**顺序不可颠倒**：推演课如果缺少情感基础，判断会流于表面。

## 4. 各学科 Kolb 四阶段适配

### 4.1 具体体验（LARP 沉浸课）

| 学科 | 体验场景 | 学生扮演 | 核心体验 |
|------|---------|---------|---------|
| 医学 | 医院/病房 | 医生、护士、患者家属 | 面对有限医疗资源的分配抉择 |
| 法学 | 法庭 | 法官、律师、当事人 | 面对疑罪案件的判断 |
| 农学 | 田间/农村 | 农技员、农民、村干部 | 面对新品种推广的信任抉择 |
| 工学 | 工地/工厂 | 工程师、工人、安全员 | 面对安全隐患的停工抉择 |
| 商学 | 会议室/市场 | 企业家、员工、消费者 | 面对商业伦理的利润抉择 |
| 教育学 | 学校/社区 | 教师、学生、家长 | 面对教育公平的选择 |
| 马克思主义理论 | 历史现场 | 历史人物、普通群众 | 面对历史转折点的个人选择 |
| 理学 | 实验室 | 研究者、审稿人、公众 | 面对科研伦理的抉择 |
| 艺术学 | 工作室/舞台 | 艺术家、策展人、观众 | 面对艺术创新的突破抉择 |

### 4.2 反思观察（间隔消化期）

- **时间**：1 周（LARP 课后到推演课前）
- **AI 推送内容**：
  - 第 1 天：回顾角色经历，提示反思
  - 第 3 天：推送 1-2 条相关拓展资料
  - 第 5 天：预告推演课视角切换
- **原则**：不评判、不引导、只提问

### 4.3 抽象概念化（推演课）

从 LARP 的"经历者视角"升级到"决策者视角"：
- LARP 中学生是"受影响者" → 推演中学生是"影响者/决策者"
- LARP 中的选择是"个人选择" → 推演中的选择是"公共决策"

### 4.4 主动实践（反思日记 + 课程应用）

- 反思日记：200-300 字，AI 辅助分析
- 课程应用：将判断转化为具体的认知框架
- 教学数据：自动关联两堂课的行为数据

## 5. 16 周学期排布模板

```
第 1-2 周：配对组 1（LARP + 推演）
第 3-4 周：配对组 2
第 5-6 周：配对组 3
第 7-8 周：配对组 4
第 9 周：期中总结
第 10-14 周：深化/自由选题/成果准备
第 15 周：成果展示
第 16 周：考核
```

### 单主题课程排布

```
┌─────────────────────────────────────┐
│  第 1 次课：LARP（2课时，~70min）     │
│  · 课前预习（AI 推送背景材料）         │
│  · 课中：5幕沉浸体验                  │
│  · 课后：AI 推送反思提示              │
├─────────────────────────────────────┤
│  间隔消化期（1周）                    │
│  · AI 自动推送反思材料               │
├─────────────────────────────────────┤
│  第 2 次课：推演（1-2课时，~55min）   │
│  · 课中：7步推演流程                 │
│  · 课后：反思日记 + AI 报告           │
└─────────────────────────────────────┘
```

## 6. 配对设计原则

### 视角递进

| 维度 | LARP | 推演 |
|------|------|------|
| 视角 | 经历者/当事人 | 决策者/旁观者 |
| 情感 | 代入、共鸣 | 理性、辨析 |
| 选择 | 个人选择 | 公共决策 |
| 理论 | 隐性体验 | 显性理论 |

### 数据连接

LARP 课数据自动传递到推演课：
- AI 可引用学生上节课的角色、选择、行为
- 推演课导入时建立对比："上一堂课你扮演 X，今天你是 Y"
- 两堂课的选择差异形成教学数据

## 7. 实验室使用场景

同一硬件，两种模式切换（~5分钟）：

| 组件 | LARP 模式 | 推演模式 |
|------|----------|---------|
| 六边形桌 | 5-6人主角 | 全班独立操作 |
| 大屏 | 剧情画面 | 投票/推演结果 |
| 终端 | 角色卡+线索 | 资料包+投票 |
| VR | 沉浸场景（可选） | 场景回顾（可选） |
| AI | 多角色对话 | 旁白+追问+推演 |
