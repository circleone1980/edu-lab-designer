# 数智实验室方案设计 Skill

为高校任意学科设计基于 Kolb 体验式学习模型的 AI/AR/VR 沉浸式教学实验室方案。

## 输入与输出

**输入**：学科 + 学校名称 + 地区 + 目标课程（+ 可选细节）

**输出**（三套交付物，共 27 个文件）：

| 交付物 | 文件数 | 详见 |
|--------|--------|------|
| ① PPT 概念方案 | 7 | `references/ppt-deliverables.md` |
| ② 建设方案 | 7 | `references/proposal-deliverables.md` |
| ③ 典型场景设计 | 13 | `references/scenario-deliverables.md` |

**集成 Skill**：`gemini-image-gen`（视觉素材生成）、`gemini-cn-prompt-patterns`（提示词工程）

---

## 双模式运行

**模式 A：交互式（默认）** — 每阶段输出后暂停，等用户审查修改后确认才进入下一阶段。

**模式 B：全自动（成熟后）** — Phase 0 确认后一键执行 Phase 1-5，输出全套材料。
<!-- EP-9: 全自动模式，Phase 0 确认后一键输出全套27个文件 -->

**交互规则**：
1. 输出初稿后暂停，询问"以上内容是否需要修改？"
2. 用户可提出修改意见，修改后重新呈现
3. 用户确认满意后才进入下一阶段
4. 用户可随时说"回到 Phase X"重新修改已完成的阶段
5. **禁止**：未经用户确认就自动进入下一阶段

## Phase 1：调研发现

> 详见 `references/research-guide.md`

### 调研维度（6 个）

1. **学校背景**：办学特色、校训、优势学科
2. **区域历史**：重大事件、重要人物、文化特色
3. **学科本地连接**：该学科在本地区的独特故事/案例
4. **校友/名人**：该校该学科走出的杰出人物
5. **教学痛点**：该学科教学的共性问题
6. **政策机遇**：教育部/省级相关政策

### 搜索策略

使用 `scripts/research-prompt-gen.js` 生成搜索查询，或手动执行三轮搜索（宏观→深度→验证）。

## Phase 2：课程架构

> 详见 `references/kolb-model.md`

### Kolb 四阶段在本学科的适配

| Kolb 阶段 | 教学实现 | 时长 |
|-----------|---------|------|
| 具体体验 | LARP 沉浸课（学生扮演角色） | 2 课时（~70min） |
| 反思观察 | 间隔消化期（AI 推送反思提示） | 1 周 |
| 抽象概念化 | 推演课（学生以自身身份判断） | 1-2 课时（~55min） |
| 主动实践 | 反思日记 + 课程应用 | 课后 |

### 关键原则：先 LARP 后推演

先体验建立情感记忆，再理性判断。顺序不可颠倒——推演课如果缺少情感基础，判断会流于表面。

### 16 周学期排布模板

```
第 1-2 周：配对组 1（LARP + 推演）
第 3-4 周：配对组 2
第 5-6 周：配对组 3
第 7-8 周：配对组 4
第 9 周：期中总结
第 10-14 周：深化/自由选题/成果准备
第 15 周：成果展示
第 16 周：考核
```

### 交互检查点

**暂停**。呈现课程架构，询问用户确认。

## Phase 3：场景创建

> 详见 `references/larp-template.md`、`references/deduction-template.md`、
> `references/role-card-design.md`、`references/scenario-deliverables.md`

这是核心阶段。为每对 LARP + 推演创建完整脚本。

### LARP 剧本（5 幕结构，~70 分钟）

```
第0步: 准备(5min) → 第1幕: 引入(15min) → 第2幕: 冲突(20min)
→ 第3幕: 抉择(10min) → 第4幕: 跨越时空的信(10min) → 第5幕: 复盘(10min)
```

每个剧本包含：设定 → 物理场景 → 5张角色卡 → AI角色 → 详细交互 → 线索表 → 投票设计 → 理论映射 → 技术实现表。

### 推演脚本（7 步流程，~55 分钟）

```
第0步: 导入(5) → 第1步: 信息收集(10) → 第2步: 投票(5)
→ 第3步: 论证(10) → 第4步: 修正(5) → 第5步: AI后果推演(10)
→ 第6步: 真相揭示(5) → 第7步: 复盘(10)
```

每个推演包含：设定 → 物理场景 → 7步交互 → 三视角资料包 → AI追问策略 → 后果推演 → 理论映射 → 技术实现表。

### 配对设计原则

- **共享主题**：LARP 和推演围绕同一历史/现实事件
- **视角递进**：LARP 是"经历者"，推演是"决策者"
- **数据连接**：LARP 课数据自动传递到推演课，AI 可引用学生上节课的行为

### 交互检查点

**每对单独确认**。完成一对 LARP+推演后暂停，询问用户满意后才开始下一对。

## Phase 5：方案生成

> 详见 `references/ppt-deliverables.md`、`references/proposal-deliverables.md`、`references/tech-mapping.md`

按顺序生成三套交付物：

### 生成顺序

```
Phase 5.1: 典型场景（13文件）— 已在 Phase 3-4 完成，此时组装目录结构
Phase 5.2: 建设方案（7文件）— 基于场景设计+技术架构+预算
Phase 5.3: PPT 概念方案（7文件）— 基于建设方案提炼精华
```

### ③ 典型场景设计（13 文件）

```
③-典型场景/
├── 配对课程设计方案.md
├── [LARP名称]/
│   ├── 00-公共设计/物理场景总述.md
│   ├── 场景1/剧本设计.md
│   ├── 场景2/剧本设计.md
│   ├── 场景3/剧本设计.md
│   └── 场景4/剧本设计.md
└── [推演名称]/
    ├── 00-公共设计/推演机制总述.md
    ├── 场景1/推演设计.md
    ├── 场景2/推演设计.md
    ├── 场景3/推演设计.md
    └── 场景4/推演设计.md
```

### ② 建设方案（7 文件）

```
②-建设方案/
├── 08-整体建设详细方案.md（13章，内部策略版，每章含"五维价值"）
├── 09-报价方案.md（6建设包 A-F，百分比预算）
├── 10-内部对照底稿.md
├── 11-建设方案完整版.md（11章，客户端，含Mermaid+效果图）
├── 12-效果图脚本与提示词.md（统一风格+逐张提示词）
├── 13-建设方案正式版.md（最终整合版）
└── 14-执行计划.md（任务追踪）
```

关键模式：
- **"五维价值"**：教学/教研/品牌/区域辐射/长期资产
- **"能力包"预算**：按功能（非硬件/软件）组织
- **两阶段实施**：阶段一（0-4月快速见效）+ 阶段二（5-12月全面部署）
- **范围边界表**：含预算内/不含/可选扩展

### ① PPT 概念方案（7 文件）

```
①-PPT概念方案/
├── 00-方案源文档.md（~500行原始材料）
├── 01-PPT逐页大纲.md（16页骨架）
├── 02-PPT逐页详细描述.md（每页4部分：目标/文案/视觉/演讲提示）
├── 03-风格一致性约束.md（10部分品牌指南）
├── 04-PPT生成提示词.md（3层策略）
├── 05-逐页讲解稿.md（朗读版）
└── 06-现场汇报强化稿.md（实战版）
```

16页叙事弧线：
```
P1:封面 → P2:为什么做 → P3:需求 → P4:组织逻辑
→ P5-P7:方向一(沉浸场景) → P8-P9:方向二(AI中台) → P10:方向三(平台)
→ P11:综合 → P12:架构 → P13:空间 → P14:实施 → P15:预算 → P16:成果
```

## 通用学科适配指引

> 详见 `references/discipline-adapter.md`

### 四维度分析框架

| 维度 | 分析内容 | 示例 |
|------|---------|------|
| 体验场景类型 | 实验室/临床/田野/法庭/工厂/社区 | 医学→临床，法学→法庭 |
| 核心冲突类型 | 伦理/效率/公平/安全/创新/传承 | 医学→伦理，工学→安全 |
| 知识形态 | 理论/技能/态度/判断/创造 | 法学→判断，农学→技能 |
| 价值教育方向 | 职业伦理/社会责任/家国情怀/创新/人文 | 工学→安全责任，教育学→人文关怀 |

### 主题发现方法论

**LARP 主题**：列出学科重大争议事件 → 筛选有"冲突+选择"的 → 确认可转5幕 → 检查本地连接
**推演主题**：从配对 LARP 提取核心两难 → 升级为决策者视角 → 设计三视角资料包

## 质量检查清单

每个剧本/推演交付前必须通过：

- [ ] LARP 包含完整 5 幕结构（第0步+第1-5幕）
- [ ] 每个角色卡包含 200+ 字背景 + 秘密 + 内在冲突

- [ ] 推演有三视角资料包（A方/B方/第三方）
- [ ] 理论映射表完整（每个关键情节都有对应理论）
- [ ] 技术实现表完整（每个教学步骤都有对应技术）
- [ ] 本地/区域元素融入（至少 3 处）
- [ ] 配对逻辑清晰（LARP 和推演的递进关系）

每套交付物交付前：

- [ ] PPT 方案：7 文件齐全，16 页结构完整
- [ ] 建设方案：7 文件齐全，预算表含范围边界
- [ ] 典型场景：13 文件齐全，配对逻辑自洽
